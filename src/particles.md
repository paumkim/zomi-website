# Zomi Particles & Word Structure

## What is a Particle?

A **particle** is a small word that has no meaning by itself — it only adds grammatical information to the word it attaches to. Think of them like English `'s` (possessive) or `-ing` (progressive) — they don't stand alone.

Zomi particles are the **glue** that makes the language work.

---

## Complete Particle Reference

### Prefixes (merge RIGHT → to the following word)

| Prefix | Meaning | Example |
|--------|---------|---------|
| `ka-` | I, my (1st person singular) | `ka paai` → I go |
| `na-` | you, your (2nd person singular) | `na paai` → you go |
| `a-` | he/she, his/her (3rd person) | `a paai` → he/she goes |
| `i-` / `ih-` | we, our (1st person plural) | `i paai` → we go |
| `ki-` | reflexive, middle voice | `ki paai` → go oneself |

`ki-` is special — it turns a verb into a middle/reflexive form:

| Without `ki-` | With `ki-` |
|---------------|------------|
| `khawm` (gather) | `kikhawm` (gather together / oneself) |
| `cing` (full/complete) | `kicing` (be filled) |
| `sim` (read/count) | `kisim` (be read/be counted) |
| `neih` (have) | `kineih` (be had / exist) |
| `paai` (go) | `kipaai` (go oneself / depart) |

**Note:** `ka-`, `na-`, `a-`, `i-` are usually written **separate** from the verb (as two words).  
`ki-` is always written **merged** to the verb.

### Suffixes (merge LEFT ← to the preceding word)

| Suffix | Meaning | Example |
|--------|---------|---------|
| `-te` | plural marker | `mi` + `te` → `mite` |
| `-ding` | future / will | `paai` + `ding` → `paaiding` |
| `-ta` | already / completed | `om` + `ta` → `omta` |
| `-ah` | in / at / on (locative) | `khawm` + `ah` → `khawmah` |
| `-in` | by / with (instrumental) | `thei` + `in` → `theiin` |
| `-'n` | subject marker (agentive) | `Zeisu` + `'n` → `Zeisu'n` |
| `-pa` | father / male | `Topa` from `To` + `pa` |
| `-pi` | big / large / mother | `tuipi` from `tui` + `pi` |
| `-cing` | completeness | `kicing` from `ki` + `cing` |
| `-gah` | berry / hanging fruit | `singgah` from `sing` + `gah` |
| `-la` | song | `Zola` from `Zo` + `la` |

---

## The Merge Direction Rule

This is the single most important rule in Zomi spelling:

> **Prefixes grab the word AFTER them (→).**  
> **Suffixes grab the word BEFORE them (←).**

```
    ki    →    cing    →    kicing
   (prefix grabs right)

    paai  ←    ding    →    paaiding
   (suffix grabs left)

    ki    →    khawm   ←    ding   →    kikhawmding
   (prefix right + suffix left = chain)
```

---

## Why Hyphens Help

When a suffix attaches to a **proper name** (person or place), keep the hyphen:

```
David'pa     → David's father  (not Davidpa — hard to read)
David'ta     → David's son     (not Davidta — confusing)
Egypt-ah     → in Egypt        (not Egyptah — looks wrong)
Bethlehem-a  → to Bethlehem    (not Bethlehema — merged)
```

The hyphen tells the reader: "this suffix belongs to the name, but they're separate words."

---

## Reduplication — Repeating Words

When a word repeats for emphasis or duration, **merge the repeats**:

| Before | After | Meaning |
|--------|-------|---------|
| `sem sem` | `semsem` | doing and doing (continuous) |
| `pai pai` | `paipai` | going on and on |
| `tawm tawm` | `tawmtawm` | a little bit |
| `mahn mah` | `mahmah` | very much |

This is like English "more and more" or "on and on."

---

## The Chaining Effect

Particles stack naturally. Each one attaches to the previous result:

```
Step 1:  ki + khawm      → kikhawm     (oneself + gather)
Step 2:  kikhawm + ding  → kikhawmding  (will gather)
Step 3:  kikhawmding + in → kikhawmdingin (by gathering)

→ kikhawmdingin
→ "by (our) gathering together"
```

```
Step 1:  mi + te        → mite      (people)
Step 2:  mite + 'n      → mite'n    (people - subject)
Step 3:  mite'n + hong it → mite'n hong it (people love us)

→ Mite'n hong it hi.
→ "People love us."
```
